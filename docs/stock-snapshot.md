# GULI Stock Snapshot Engine

## 資料來源與範圍

個股快照只使用 `public/data/twse-stocks/latest.json` 中，通過驗證且 `instrumentType = stock` 的 TWSE 官方上市股票盤後資料。ETF、ETN、權證、unknown 與 invalid 記錄均排除。資料為盤後快照，不是即時行情。

## 評分規則

- 價格強度：當日漲跌幅 45%、收盤在高低區間的位置 35%、收盤相對開盤 20%。
- 流動性：同一交易日所有有效股票的成交金額 45%、成交量 35%、成交筆數 20% 百分位。
- 估值風險：只依官方本益比採一般性級距（15、25、40、60 倍）。這不是產業估值比較；本益比缺值保持 `null`。
- 快照分數：價格強度 45%、流動性 35%、估值安全 20%，其中估值安全為 `100 - 估值風險`。本益比缺值時以其餘可用權重重新正規化；價格強度或流動性不足時總分為 `null`。

所有分數限制在 0–100。此分數與既有健康分數不同，沒有使用 RSI、波動率、法人或模擬歷史。

## 風險門檻

- 單日漲幅 ≥ 7%、單日跌幅 ≤ -7%。
- 當日振幅 ≥ 8%。
- 收盤位置 ≤ 20%。
- 流動性分數 ≤ 15。
- 本益比 ≥ 60 倍（一般性提示，不做產業結論）。
- OHLC、PE 或官方 partial 狀態會產生相應缺值／品質提示。

## JSON 與歷史

每日索引位於 `public/data/stock-history/YYYY-MM-DD/index.json`，單股檔位於 `stocks/<symbol>.json`。`latest.json` 只含排行所需摘要，避免首頁下載全部個股明細。相同日期重跑會覆寫該日期並在 archive index 去重；所有檔案採暫存檔後 rename。

Memory 僅使用實際存在的快照，不補造缺少的交易日。資料不足會回傳實際筆數與 warning。Industry Snapshot 尚無官方個股產業 mapping，因此不會以名稱猜測映射。

## 使用限制

此功能只提供資料整理與規則型觀察，不構成投資建議。尚未接入 TPEX、即時行情、官方產業 mapping、法人與長期歷史指標。
